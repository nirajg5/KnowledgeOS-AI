# KnowledgeOS AI — Frontend

An enterprise-grade, AI SaaS-style dashboard for **KnowledgeOS AI**, an Enterprise
AI Document Intelligence Platform. Built with **Streamlit**, designed to feel like
a polished product in the spirit of ChatGPT, Claude, Notion AI, and Perplexity —
dark mode, glassmorphism, gradients, and smooth interactions throughout.

## ✨ Highlights

- **Dashboard** — live KPIs, document/status/language charts, recent activity feed
- **Documents** — searchable, filterable, paginated table with preview/process/download/delete
- **Upload** — drag-and-drop ingestion with animated progress and auto-processing
- **AI Chat** — ChatGPT-style interface with chat history, streaming feel, and citation cards
- **Reports** — one-click AI report generation with live Markdown preview and history
- **Analytics** — interactive Plotly charts across time ranges
- **Settings** — backend URL, theme, and LLM generation parameters

The frontend talks to a **FastAPI** backend (documents, chat, reports, analytics)
through a single service layer (`services/api_client.py`). If the backend is
unreachable, every page **gracefully falls back to realistic demo data** so the
UI is always fully functional for demos, design review, or local development —
no crashes, no blank screens.

## 🚀 Getting Started

```bash
cd frontend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

The app opens at `http://localhost:8501`.

## 🔌 Connecting to the FastAPI Backend

By default the app points at `http://localhost:8000`. Change this any time from
**Settings → Backend Connection**, or by editing `DEFAULT_SETTINGS["backend_url"]`
in `utils/session_state.py`.

Expected backend endpoints (see project spec):

```
POST   /documents/upload
GET    /documents/
GET    /documents/{id}
PUT    /documents/{id}
DELETE /documents/{id}
GET    /documents/stats
GET    /documents/filter
GET    /documents/page
GET    /documents/search
GET    /documents/{id}/preview
GET    /documents/{id}/download
POST   /documents/{id}/process

POST   /chat
GET    /chat/{session_id}

GET    /analytics/dashboard

POST   /reports
GET    /reports
POST   /reports/generate/{document_id}
POST   /reports/generate-export/{document_id}
```

If any endpoint is unreachable or errors out at the network level, the client
in `services/api_client.py` automatically falls back to demo data generated by
`services/demo_data.py` — the UI never shows a raw exception to the user.

## 🎨 Design System

All visual tokens (colors, radii, shadows, gradients) live as CSS custom
properties at the top of `assets/style.css`. Update them there to re-theme the
entire product consistently.

## 🧩 Extending

- Add a new page: drop a file into `pages/` following the `N_emoji_Title.py`
  naming convention, then add it to `NAV_ITEMS` in `components/sidebar.py`.
- Add a new reusable UI piece: add it to `components/` and import it where needed.
- Swap demo data for live data: once your FastAPI backend implements an
  endpoint, `api_client.py` will automatically prefer it over the demo fallback.
